Includes + Routing

The minimal library to compose HTML pages and handle navigation without a framework.

FragmentJS loads HTML fragments using the fragment, shows and hides sections with the .fragment, updates history and triggers callbacks for dynamic data.

Focus

Size ~5 KB
Dependencies jQuery only
Setup 2 min

An ultra-light layer on top of HTML and jQuery to build small SPAs, internal dashboards, or technical microsites with full control over markup.

Zero build Legacy friendly Micro frontends

Central config

Define routes, menu and callbacks in fragment.config.js to control the flow.

Modular sections

Each page is an HTML section with a unique id and the fragment.

Instant refresh

Force the current route with FragmentJS.refresh() to realign UI and data.

Typical use cases

  • Internal dashboards fast routing
  • Technical microsites modular includes
  • Legacy tooling jQuery ready
  • Rapid prototypes zero build

Why FragmentJS

Keep full control over HTML, CSS, and assets. No bundler, no heavy runtime. Just a minimal core and a central config.

Lightweight routing Data hooks History API Modular sections

Quick start

script order 
<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.7.0/jquery.min.js"></script>
<script src="lib/js/fragment/fragment.config.js"></script>
<script src="lib/js/fragment/fragment.js"></script>

Enable debug: true to bust include cache during development.

Ideal workflow

  1. 1. Define routes and menu.
  2. 2. Create standalone HTML sections.
  3. 3. Aggancia Data hooks con onNavigate.
  4. 4. Navigate with FragmentJS.navigateTo().
