Practical guide

Build your micro-site in a few steps.

FragmentJS works on plain HTML: separate layout and sections, then let the router show only what is needed.

1. Import the scripts

Keep the order: jQuery, config, core.

<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.7.0/jquery.min.js"></script>
<script src="lib/js/fragment/fragment.config.js"></script>
<script src="lib/js/fragment/fragment.js"></script>

2. Configure the routes

Decide which sections to show for each path.

window.FragmentJSConfig = {
  fallbackRoute: "/",
  menuItem: ["/", "/Guide", "/api"],
  routes: [
    { path: "/", section: "section-overview", menu: "/" },
    { path: "/Guide", section: "section-guide", menu: "/Guide" },
    { path: "/api", section: "section-api", menu: "/api" }
  ]
};

3. Add the sections

Each section is an HTML file with the fragment and a unique id. 

<section fragment="/sections/overview.html"></section>
<section fragment="/sections/guide.html"></section>

4. Control the UI

Use navigateTo to change pages and refresh to realign the UI. 

FragmentJS.navigateTo("/api");
FragmentJS.refresh(200);

File map

Organize the project by sections and config, so each page is isolated. 

public/
├─ index.html
├─ sections/
│  ├─ overview.html
│  ├─ guide.html
│  ├─ api.html
│  └─ ...
└─ lib/
   └─ js/fragment/
      ├─ fragment.js
      └─ fragment.config.js

Pre-launch checklist

  • Fallback route configured.
  • Each section has an id and fragment class.
  • Menu matches menuItem.
  • Debug disabled in production.

Data hooks e menu

In onNavigate you can load data, highlight the menu, or update the header. 

onNavigate: function (payload) {
  fragmentCallback(payload.menuIndex);
  setHeaderAction(payload.menuIndex);
}

Debug mode

When debug is enabled, FragmentJS appends a timestamp to includes to avoid cache. 

window.FragmentJSConfig = {
  debug: true
};
