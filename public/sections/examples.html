Here you will find typical setups: static pages, dynamic routes, and data loading with callbacks.

Dynamic routes

Use params for detail pages. In the payload you will find params. 

routes: [
  { path: "/utenti", section: "section-utenti" },
  { path: "/utenti/:id", section: "section-dettaglio-utente" }
]
params.id path matcher

Modular includes

Each section lives in a dedicated HTML file, reusable and readable. 

<section fragment="/sections/dashboard.html"></section>
<section fragment="/sections/utenti.html"></section>
Tip: use names consistent with section-*.

Data hooks custom

Usa onNavigate to initialize charts, fetches, or tables. 

onNavigate: function (payload) {
  if (payload.path === "/utenti") {
    window.loadLogtoUsers?.();
  }
  if (payload.path.startsWith("/utenti/")) {
    window.loadLogtoUserByPath?.(payload.path);
  }
}

CTA + navigation

Buttons can change routes without reloading the page. 

<button onclick="FragmentJS.navigateTo('/api')">
  Vai alle API
</button>
